

## Problem
Word-tap translations (in transcripts, Discover videos, Souq News, Reading Practice, etc.) are generated by AI **without any sentence context**. The model sees only the bare Arabic word, so it returns the most common dictionary meaning — which often doesn't match the actual usage in the sentence. Sentence translations are correct because they're produced as a whole unit.

## Fix Strategy
Pass the parent sentence (Arabic + its English translation) into the word/phrase translation calls so the AI can pick the meaning that fits the context. Apply this everywhere — dialect-agnostic — so all modules benefit.

## Changes

### 1. Edge function: `supabase/functions/translate-phrase/index.ts`
- Accept new optional fields: `sentenceArabic`, `sentenceEnglish`, `dialect`.
- When provided, inject them into both prompts:
  - Translation prompt: "Here is the full sentence and its accepted English translation. Translate the highlighted word/phrase in a way that fits this exact context."
  - MSA prompt: same context, return MSA equivalent matching that usage.
- Backward compatible — works with no context too.

### 2. Edge function: `supabase/functions/word-enrichment/index.ts`
- Accept optional `sentenceArabic` and `sentenceEnglish`.
- Update the system+user prompt to: "Define this word as it is used in the given sentence. Prefer the sense that matches the sentence's English translation."
- Keep root + uses generic; only `definition` is context-aware.

### 3. Frontend callers — pass sentence context

| File | Call site | Add context |
|---|---|---|
| `src/components/transcript/LineByLineTranscript.tsx` | single-word `translate-phrase` (line ~101) | `sentenceArabic: parentLine.arabic`, `sentenceEnglish: parentLine.translation`, `dialect` |
| `src/components/transcript/LineByLineTranscript.tsx` | compound `translate-phrase` (line ~482) | same — uses `line.arabic`/`line.translation` |
| `src/pages/DiscoverVideo.tsx` | `translate-phrase` (line ~72) | pass `parentLine.arabic` + `parentLine.translation` + dialect |
| `src/components/shared/TappableArabicText.tsx` | `word-enrichment` (line ~30) | accept new prop `sentenceContext?: { arabic; english }` and forward it; consumers (Souq News, Reading Practice, etc.) pass the surrounding sentence when available |

### 4. Consumers of `TappableArabicText`
Search and update callers in `SouqNews`, `ReadingPractice`, `BibleReading`, etc. to pass the sentence the word came from. Where the component is used on standalone phrases (no sentence), behaviour stays unchanged (falls back to dictionary meaning).

## Result
- Tap a word in a transcript → AI sees the line + its translation → returns the contextual gloss (e.g. "bank" as institution vs riverbank).
- Saving as a flashcard captures that contextual meaning, so review cards are accurate too.
- Works identically for Gulf, Egyptian, Yemeni — dialect is already in scope at every call site.
- Backward compatible: omitting context still works.
